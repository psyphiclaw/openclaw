## 1. 项目背景与动机

### 1.1 行业痛点

人类行为研究正处于"数据爆炸、洞察稀缺"的悖论中：

- **数据孤岛**：一场典型的人因实验可能同时采集面部表情（FaceReader）、眼动（Tobii/EyeLink）、脑电（BrainVision/EGI）、生理信号（Biopac EDA/ECG/EMG）、近红外（fNIRS）等 5 种以上模态数据，但研究者需要在不同软件中逐一分析，难以跨模态关联。
- **分析门槛高**：MNE-Python、EEGLAB、FieldTrip 等工具功能强大但学习曲线陡峭，心理学研究者往往需要数月才能上手。
- **融合洞察缺失**：现有工具均为单模态分析，"看到面孔皱眉的同时心率升高、注意力从目标区域偏移"这类多模态关联洞察完全依赖人工经验判断。
- **成本高昂**：商业软件授权费用动辄数万至数十万美元，中小型实验室和研究团队难以承受全套方案。

### 1.2 我们的动机

PsyPhiClaw 诞生于一个朴素的愿景：**让每一位心理学和神经科学研究者，都能像和同事对话一样完成多模态数据分析**。

我们的灵感来源于 [EthoClaw](https://github.com/penciler-star/EthoClaw)——一个基于 OpenClaw 的动物行为分析开源平台。EthoClaw 证明了 AI Agent 可以大幅简化行为学研究工作流。我们将其核心理念扩展到人类行为研究领域，并引入**多模态融合洞察**这一突破性能力。

### 1.3 项目愿景

> 构建全球首个开源的、AI 驱动的人类多模态行为洞察平台，让跨模态数据分析从"专家专属"变为"人人可用"。

### 3.1 竞品全景图

| 维度 | Noldus 矩阵 | EthoClaw | MNE-Python | Tobii Pro Lab | Biopac AcqKnowledge |
|------|------------|----------|------------|---------------|---------------------|

| **模态** | 表情+行为+音频 | 动物轨迹 | EEG/MEG | 眼动 | EDA/ECG/EMG |
| **AI能力** | 规则引擎 | LLM Agent | 无 | 基础统计 | 基础统计 |
| **开源** | ❌ | ✅ | ✅ | ❌ | ❌ |
| **多模态融合** | Observer XT (有限) | ❌ | ❌ | ❌ | ❌ |
| **自然语言交互** | ❌ | ✅ | ❌ | ❌ | ❌ |
| **价格** | $10K-100K+/年 | 免费 | 免费 | $5K-30K | $8K-50K |

### 3.2 详细分析

#### Noldus 产品矩阵
- **优势**：行业标杆，Observer XT 提供多模态数据对齐，FaceReader 表情识别精度高
- **劣势**：封闭生态、授权费用高、多模态融合仅限时间对齐，缺乏 AI 驱动的关联洞察
- **启示**：验证了多模态行为分析的市场需求，但价格门槛和创新速度是可突破的空间

#### EthoClaw（核心参考）
- **优势**：自然语言驱动、模块化 Skill 架构、基于 OpenClaw 生态
- **劣势**：仅支持动物行为，无人类多模态数据支持
- **对标**：PsyPhiClaw 是 EthoClaw 的"人类行为版"，继承其 AI Agent 架构，扩展到人类多模态数据

#### MNE-Python
- **优势**：EEG/MEG 分析的学术标准，社区活跃
- **劣势**：编程门槛高，无自然语言交互，仅限电生理
- **启示**：底层计算引擎可直接复用，在其之上构建自然语言接口

#### Tobii Pro Lab / EyeLink
- **优势**：眼动追踪硬件+软件的一体化方案
- **劣势**：仅支持眼动，且无法与其他模态的原始数据融合分析

#### Biopac AcqKnowledge
- **优势**：生理信号采集与分析的专业工具
- **劣势**：生态封闭，数据分析功能偏基础

### 3.3 竞争优势总结

**PsyPhiClaw 的核心差异化 = EthoClaw 的 AI Agent 体验 + Noldus 的多模态覆盖 + 市场首创的跨模态融合洞察**

| 能力 | 我们 | 最近竞品 |
|------|------|----------|
| 自然语言数据分析 | ✅ | EthoClaw（动物） |
| 多模态数据支持 | 5种模态 | Noldus（商业，3种核心） |
| 跨模态融合洞察 | ✅ 首创 | **无同类产品** |
| 开源免费 | ✅ | MNE-Python（单模态） |
| 论文级图表生成 | ✅ | EthoClaw（动物） |

## 4. 产品定位与差异化

### 4.1 一句话定位

> **PsyPhiClaw：对话式人类多模态行为洞察平台**
> 用自然语言完成面部表情、眼动、脑电、生理信号、近红外等多模态数据的分析、融合与洞察提取。

### 4.2 三大核心差异化

#### ① 对话式分析（Conversational Analysis）
不写代码、不学工具。研究者只需描述需求，AI Agent 自动调用对应 Skill 完成分析：
> "帮我把这场可用性测试中的面部表情和心率数据对齐，找出任务难度最高的3个时刻。"

#### ② 多模态融合洞察（Multimodal Fusion Insight）
**市场首创**。不是简单的时间对齐，而是 AI 驱动的跨模态关联分析：
- 情绪-生理联动：面孔皱眉 + 皮电升高 → 检测到"认知冲突+情绪唤醒"
- 注意-认知联动：注视偏离目标 + P300振幅降低 → 检测到"注意力分散"
- 生成人类可读的洞察报告

#### ③ 开源生态（Open Ecosystem）
- 基于 OpenClaw 框架，兼容其全部插件和通道
- MIT 协议开源，社区共建
- Skill 模块化，研究者可自定义扩展

### 4.3 产品哲学

| EthoClaw（动物） | PsyPhiClaw（人类） |
|-------------------|---------------------|
| 动物中心检测 | 面部表情识别 |
| 动物姿态估计 | 眼动追踪分析 |
| 轨迹热力图 | 脑电事件相关电位 |
| 速度/距离参数 | 生理信号特征提取 |
| 多参数统计图 | 多模态融合洞察 |
| 分析报告生成 | 洞察报告生成 |
| 本地知识库（文献） | 本地知识库（文献） |
| 每日论文推送 | 每日论文推送 |

## 5. 功能矩阵

### 5.1 Skill 总览（14个核心 Skill）

我们将功能拆分为 14 个模块化 Skill，覆盖数据采集、预处理、分析、可视化和 AI 洞察全链路：

#### 📊 数据接入类（5个）

| # | Skill 名称 | 功能描述 |
|---|-----------|----------|
| 1 | `psyphiclaw-face-import` | 导入 FaceReader CSV 数据，解析 Action Unit、基本情绪（Valence/Arousal）、头部朝向等 |
| 2 | `psyphiclaw-eye-import` | 导入 Tobii TSV / EyeLink ASC 数据，解析注视点、扫视、瞳孔直径、AOI 等 |
| 3 | `psyphiclaw-eeg-import` | 导入 BrainVision / EGI / EDF 格式，解析 EEG 通道数据、事件标记、采样率等 |
| 4 | `psyphiclaw-physio-import` | 导入 Biopac AcqKnowledge / exported CSV 数据，解析 EDA、ECG、EMG、呼吸、皮温等 |
| 5 | `psyphiclaw-fnirs-import` | 导入 fNIRS 光学密度数据（Homer3/NIRX 格式），解析含氧/脱氧血红蛋白浓度 |

#### 🔧 预处理类（3个）

| # | Skill 名称 | 功能描述 |
|---|-----------|----------|
| 6 | `psyphiclaw-preprocess-face` | FaceReader 数据清洗：插值缺失帧、平滑滤波、异常值检测、时长截取 |
| 7 | `psyphiclaw-preprocess-eye` | 眼动数据清洗：眨眼插值、漂移校正、低质量采样过滤、注视/扫视分类 |
| 8 | `psyphiclaw-preprocess-eeg` | EEG 预处理：通道选择、滤波（带通/带阻/陷波）、ICA 去伪迹、重参考、分段 |
| 9 | `psyphiclaw-preprocess-physio` | 生理信号预处理：ECG R峰检测、EDA 峰值检测、EMG 整流平滑、信号去噪 |
| 10 | `psyphiclaw-preprocess-fnirs` | fNIRS 预处理：运动伪迹校正（Spline/波纹滤波）、Beer-Lambert 转换、基线校正 |

#### 🧠 分析与可视化类（5个）

| # | Skill 名称 | 功能描述 |
|---|-----------|----------|
| 11 | `psyphiclaw-eeg-analysis` | ERP/ERD/ERS 分析、时频分析、频谱功率、功能连接、溯源估计 |
| 12 | `psyphiclaw-eye-analysis` | 热力图生成、扫视路径图、AOI 统计、瞳孔响应分析、阅读模式分析 |
| 13 | `psyphiclaw-physio-analysis` | HRV 分析（时域/频域/非线性）、EDA 响应分析、EMG 活动量分析、压力指数计算 |
| 14 | `psyphiclaw-visualization` | 论文级多模态图表生成：时间轴对齐图、热力图、小提琴图、雷达图、聚类图、论文 Figure 布局 |

#### 🤖 AI 洞察类（2个）

| # | Skill 名称 | 功能描述 |
|---|-----------|----------|
| 15 | `psyphiclaw-fusion-insight` | **核心差异化 Skill**：跨模态时间对齐 + 关联分析 + 异常检测 + LLM 洞察报告生成 |
| 16 | `psyphiclaw-report` | 自动生成完整分析报告（含实验背景、方法、结果、结论），支持 PDF/Markdown 输出 |

> **注**：上文列出 16 个描述，其中部分预处理类 Skill 可合并。最终交付为 **14 个 Skill**，具体合并策略见 `ARCHITECTURE.md`。

#### 📚 辅助类（沿用 EthoClaw 经验）

| # | Skill 名称 | 功能描述 |
|---|-----------|----------|
| - | `psyphiclaw-literature-search` | 每日 arXiv/PubMed 关键词论文推送，自然语言文献检索 |
| - | `psyphiclaw-pdf-research` | 本地 PDF 论文阅读、摘要生成、关键信息提取 |

> 辅助类 Skill 沿用 EthoClaw 的成熟实现，优先级较低。

## 7. 技术选型

| 层次 | 技术选型 | 理由 |
|------|----------|------|
| **AI Agent 框架** | OpenClaw (fork) | 成熟的 Agent 框架，Skill 模块化架构，支持自然语言交互 |
| **EEG 分析** | MNE-Python | 学术标准，功能全面（预处理、ERP、时频、溯源），社区活跃 |
| **生理信号分析** | NeuroKit2 | EDA/ECG/EMG 信号处理的一站式方案，API 友好 |
| **眼动分析** | PyGaze + 自研 | PyGaze 提供基础支持，AOI 分析和热力图自研增强 |
| **fNIRS 分析** | MNE-NIRS | MNE 生态的 fNIRS 扩展，与 EEG 分析统一框架 |
| **数据可视化** | Plotly (交互) + Matplotlib (论文) | Plotly 支持交互式探索，Matplotlib 生成论文级矢量图 |
| **数据结构** | Pandas + Xarray | Pandas 处理表格数据，Xarray 管理多维神经数据 |
| **时间同步** | LSL (Lab Streaming Layer) | 实时流同步标准；支持 TTL + 手动标记回退 |
| **LLM** | OpenClaw 配置模型 | 支持 GLM-4、GPT-4o、Claude 等多模型切换 |
| **运行环境** | Python 3.11+ / Conda | 科学计算生态完善，依赖隔离 |
| **文档/部署** | Docker + GitHub Actions | 标准化部署，CI/CD 自动测试 |

## 10. 风险评估与应对

| # | 风险 | 等级 | 影响 | 应对策略 |
|---|------|------|------|----------|
| R1 | **数据格式碎片化**：各厂商私有格式，解析工作量大 | 高 | 延误 Phase 0-1 | 优先支持主流格式（BrainVision、Tobii TSV、Biopac CSV），通过社区 PR 扩展 |
| R2 | **时间同步精度不足**：不同设备采样率差异大，同步误差影响融合分析 | 高 | 融合洞察准确性 | 实现 LSL + TTL 双通道同步，提供精度评估工具 |
| R3 | **LLM 洞察幻觉**：AI 生成的洞察可能不准确或误导 | 高 | 用户信任度 | 设计严格护栏：仅报告统计显著结果、标注置信度、支持人工审核 |
| R4 | **计算资源需求**：EEG/fNIRS 分析需要大量内存和计算 | 中 | 部署门槛 | 支持 Downsample 预处理、提供轻量模式、推荐硬件配置 |
| R5 | **用户采用门槛**：研究者不熟悉自然语言交互模式 | 中 | 用户增长 | 提供详细教程、视频 Demo、Quick Start 指南 |
| R6 | **OpenClaw 上游变更**：基础框架更新可能带来兼容性问题 | 低 | 维护成本 | 锁定版本、定期同步上游、建立 CI 兼容测试 |
| R7 | **学术认可度**：新工具需要时间获得学术社区信任 | 中 | 引用量 | 发布 preprint、邀请合作实验室试用、提供数据复现案例 |

## 11. 商业模式建议

### 11.1 核心原则：开源优先，服务增值

PsyPhiClaw 核心平台 MIT 开源，通过增值服务实现可持续运营。

### 11.2 收入模型

| 层次 | 产品 | 定价 | 说明 |
|------|------|------|------|
| **Community** | 开源平台 | 免费 | 全部 Skill 开源，社区支持 |
| **Professional** | PsyPhiClaw Pro | ¥2,000/月/席位 | 优先技术支持、私有部署指导、定制 Skill 开发 |
| **Enterprise** | 企业版 | 按项目报价 | 合规审计支持（IEC 62366）、数据安全审计、SLA 保障、驻场支持 |
| **Cloud** | 云分析服务 | ¥500/次分析 | 上传数据 → AI 分析 → 获取报告（无需本地部署） |
| **Training** | 培训认证 | ¥5,000/人/期 | 线上/线下培训班、认证考试 |

### 11.3 目标市场切入策略

```
Phase 1: 学术研究圈（免费策略，快速获取用户和引用）
    ↓
Phase 2: 人因工程咨询公司（Pro 版，解决数据量大、效率需求）
    ↓
Phase 3: 医疗器械企业（Enterprise 版，合规需求强、付费意愿高）
```

### 11.4 与 Noldus 的协同可能

作为 Noldus 内部孵化的开源项目，PsyPhiClaw 可以：
- **技术验证**：验证 AI Agent 驱动的分析模式是否可行
- **生态扩展**：作为 Noldus 商业产品的轻量级开源替代层，扩大市场覆盖
- **数据增值**：为 Noldus 云平台提供 AI 分析能力
- **品牌影响力**：提升 Noldus 在开源社区的学术品牌
